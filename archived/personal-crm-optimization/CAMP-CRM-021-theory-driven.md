# CAMP-CRM-021: CRM 理论驱动 — 人脉智能升级

**状态**: open
**优先级**: 高
**创建日期**: 2026-06-10
**描述**: 基于主流 CRM 理论框架（Dunbar 分层、BRM 七要素、弱关系理论、Habit Loop），对个人 CRM 系统进行智能化升级，从"联系人管理"进化为"人脉资产运营"

**理论依据**:
- **Dunbar 邓巴数**: 人类认知极限 ~150 个稳定关系，1459 人必须分级
- **BRM 七要素模型**: 价值 × 三面（情面/体面/场面）× 三度（频度/长度/深度）
- **弱关系强度理论 (Granovetter)**: 工作机会来自弱关系而非密友
- **Tier 分层制**: VIP / Active / At Risk / Dormant 四级
- **Habit Loop 习惯循环**: 触发→行动→奖励，降低使用摩擦

## Acceptance Criteria

### AC1: Dunbar 层级自动分类系统
- [ ] 后端新增 `dunbar_tier` 字段（T1 核心圈 / T2 密友圈 / T3 朋友圈 / T4 认识的人 / T5 名片库）
- [ ] 自动分级算法：
  - T1: `record_count > 20 且 days_since_last_contact < 7`（高频深度互动）
  - T2: `record_count > 5 且 days_since_last_contact < 30`（月度联系）
  - T3: `record_count > 0 且 days_since_last_contact < 90`（季度联系）
  - T4: `record_count > 0`（有过记录但超期）
  - T5: `record_count == 0`（从未联系）
- [ ] 新增 API `GET /api/dashboard/dunbar-stats` 返回各层级人数分布和占比
- [ ] Dashboard 新增"邓巴层级分布"可视化组件（饼图或环形图）
- [ ] 联系人详情页显示当前层级标签，支持手动调整
- [ ] 联系人列表支持按 Tier 筛选和排序
- [ ] 验证：1459 人的自动分类结果合理（T5 应占最大比例）

### AC2: 关系健康度评分（BRM 七要素雷达图）
- [ ] 后端新增 `GET /api/contacts/{id}/relationship-score` 端点
- [ ] 七维度评分算法（每项 0-100 分）：
  - **价值分**: 基于 company + title + tag_count 综合评估对方资源价值
  - **频度分**: 实际联系频率 vs target_frequency_days 的达成率
  - **长度分**: 认识天数（created_at 到今天），越长分数越高（有衰减曲线）
  - **深度分**: record_count + 平均 content 字符数（衡量交流质量）
  - **情面分**: 是否有共同标签（同事/同学/室友等）
  - **体面分**: notes 完整度 + 最近一次跟进记录质量
  - **场面分**: 暂用 tag_count 作为代理指标（后续可扩展活动记录）
- [ ] 总分 = 加权平均（权重可配置：价值 15% + 频度 25% + 长度 10% + 深度 20% + 情面 10% + 体面 10% + 场面 10%）
- [ ] 前端联系人详情页新增"关系健康度"卡片：七要素雷达图 + 总分 + 改进建议
- [ ] Dashboard 新增"关系健康 Top 10 / Bottom 10"列表
- [ ] 评分低于 40 分的联系人在待联系清单中高亮标记为"需激活"
- [ ] 验证：目标联系人中至少有 3 人评分 > 60

### AC3: 弱关系桥接发现引擎
- [ ] 后端新增 `GET /api/dashboard/weak-ties` 端点
- [ ] 弱关系价值评分算法：
  - **行业差异度**: 与用户所在公司不同则加分（迈智微电子同事间不加分）
  - **跨圈子指数**: 标签组合越独特（稀有标签）得分越高
  - **信息桥梁潜力**: 同时属于多个不同分组/标签的联系人
  - **响应率代理**: 有过记录且 mood="愉快" 的比例
- [ ] 返回 Top 20 弱关系推荐，包含：姓名、公司、桥梁理由（如"连接策堤特与中兴两个圈子"）
- [ ] 前端 Dashboard 新增"弱关系发现"卡片
- [ ] 点击可查看该联系人连接的"桥梁图谱"（简单文字描述即可）
- [ ] 弱关系推荐与"本周待联系清单"去重（已推荐的不再重复出现）

### AC4: 联系打卡与成就系统（Habit Loop Reward）
- [ ] 后端新增 contact_activity 表记录打卡行为：
  - id, contact_id, activity_type（call/wechat/meeting/email）, created_at
- [ ] 新增 API:
  - `POST /api/contacts/{id}/check-in` 记录一次打卡
  - `GET /api/dashboard/achievements` 返回成就列表和进度
- [ ] 成就体系设计：
  - 🌱 **初出茅庐**: 本周首次联系 1 人
  - 🔥 **连续打卡**: 连续 7 天每天至少联系 1 人
  - 🎯 **目标达成**: 本周完成所有目标联系人联系
  - 🌐 **广撒网**: 本周联系 10 个不同的人
  - 💪 **深度交流**: 单次联系记录超过 200 字
  - ⭐ **月度之星**: 月度累计联系 30+ 人
  - 🏆 **人脉大师**: 累计 100 次有效联系
- [ ] 前端实现：
  - 联系记录保存后自动弹出打卡成功动画（+1 徽章/烟花效果）
  - Dashboard 右上角新增"成就徽章"入口
  - 成就页面展示已获得/未获得的徽章墙
  - 本周进度条："本周已联系 X/10 人"
- [ ] 打卡数据同时写入 contact_records（保持数据一致性）

### AC5: 智能联系时机建议（AI Reminders）
- [ ] 后端新增 `GET /api/dashboard/smart-suggestions` 端点
- [ ] 智能建议规则引擎（基于业务日历 + 行为模式）：
  - **周一上午**: 推荐上周未完成的待联系人（新周开局）
  - **节假日前 3 天**: 推荐发送节日问候的联系人列表
  - **对方生日前 7 天**: 如有 birthday 字段则提前提醒（需先支持录入生日）
  - **超期预警**: 超过 target_frequency_days × 2 的人标红
  - **冷启动唤醒**: 从 T5 中随机选 2-3 人作为"本周破冰对象"
  - **行业热点**: 同行业联系人 + 近期有相关新闻时触发（可选）
- [ ] 每个建议附带推荐理由文案模板：
  - "张三已经 180 天没联系了，上次聊的是 XX 项目"
  - "李四是中兴的前同事，可以考虑约个饭"
- [ ] 前端 Dashboard 将"本周待联系清单"升级为"智能建议面板"：
  - Tab 切换：待联系 / 弱关系发现 / 智能建议
  - 每条建议带一键操作按钮（复制问候语模板 / 打开微信）
- [ ] 支持"稍后提醒"/"跳过此条"/"已完成"三种操作反馈
