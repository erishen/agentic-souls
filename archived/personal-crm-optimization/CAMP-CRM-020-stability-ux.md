# CAMP-CRM-020: 稳定性优化与体验改进

**状态**: open
**优先级**: 中
**创建日期**: 2026-06-09
**描述**: 修复全面排查中发现的 8 个问题，聚焦服务稳定性、UI一致性、用户体验

---

## Acceptance Criteria

### AC1: 后端服务稳定性排查与加固
- [ ] 排查后端频繁崩溃根因（检查 uvicorn 日志、内存占用、异常请求）
- [ ] 如确认是 --reload 触发的，考虑改用文件监控重启或增加超时配置
- [ ] 确保异常不会导致整个进程挂起（关键路径加 try/except）
- [ ] 验证：连续运行 1 小时无崩溃

### AC2: Directory 页面 UI 优化
- [ ] 为"目标联系人"分组添加专属图标（如 🎯 或 📋）
- [ ] 修正展开/折叠箭头方向：展开→▼ / 折叠→▶
- [ ] 目标联系人分组图标和颜色在 Directory 和 Contacts 页面保持一致

### AC3: "发微信"/"打电话"按钮桌面端适配
- [ ] 桌面浏览器点击"发微信"时显示 tooltip 提示"请在手机微信中打开"
- [ ] 检测平台：navigator.userAgent 不包含 Mobile 时显示提示
- [ ] "打电话"按钮同理处理

### AC4: 批量操作添加 loading 状态
- [ ] 批量删除时显示 loading 遮罩 + "正在删除 N 个联系人..."
- [ ] 批量改分组时显示 "正在修改分组..."
- [ ] 合并联系人时显示 "正在合并..."
- [ ] 操作完成后 Toast 反馈成功/失败数量

### AC5: GROUPS 常量动态化
- [ ] 新增 API `GET /api/groups` 返回数据库中所有不重复的分组名
- [ ] 前端 Contacts.tsx / ContactEditForm.tsx / Directory 页面的分组下拉从 API 获取
- [ ] 保留 api.ts 中 GROUPS 作为 fallback 默认值
- [ ] 新增分组后前端自动可用，无需硬编码更新

### AC6: 联系人详情页快捷键支持
- [ ] 按 Esc 键返回联系人列表
- [ ] 按 Alt+← 返回列表
- [ ] 编辑模式下按 Esc 取消编辑（而非返回列表）

### AC7: 搜索历史管理增强
- [ ] 搜索历史下拉框底部添加"清除搜索历史"按钮
- [ ] 清除操作需二次确认（或直接清除+Toast提示）
- [ ] 历史记录上限从 10 调整为 20 条

### AC8: 联系人列表页性能优化（1459 人场景）
- [ ] 首屏加载时间 < 2 秒（当前 50 条/页应该已满足）
- [ ] 分组筛选切换时显示 skeleton/loading
- [ ] 大数据量下表格渲染不卡顿（虚拟滚动或分页保持当前策略即可）
