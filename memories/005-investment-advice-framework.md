# Memory: 投资建议"先卖后想"模式缺陷

## 问题 ID
`MEM-005`

## 发现时间
2026-05-24

## 问题描述

### 现象
在提供投资建议时，多次出现以下循环：
1. 建议卖出某产品（理由：收益低/风险高/占比过大）
2. 用户问"卖了钱去哪"
3. 发现没有更好的去处
4. 收回建议，改为不动

具体案例：
- 博道盛彦：建议减仓3000-5000 → 用户问钱去哪 → 发现余额宝才1.5% → 改为不动
- 东方红配置精选：建议全卖 → 发现4,937放余额宝不如留着吃3-4% → 改为不动
- 国联安沪深300增强：建议减仓5000 → 用户已减3轮 → 改为不动
- 余额宝转出：建议转出13000 → 没想好转去哪 → 改为不动

### 影响
1. 建议反复修正，降低可信度
2. 用户可能因为错误建议做出不理性的操作
3. 卖出后放活期反而损失收益

### 根因分析
1. 建议逻辑是"这个不好→应该卖"，缺少"卖了之后去哪更好"的验证
2. 只看了当前产品的缺点，没有对比替代方案的收益
3. 忽略了"不动"本身也是一种选择

## 解决方案

### 投资建议决策框架

```
Step 1: 识别问题产品
  → 这个产品收益低/风险高/占比过大？

Step 2: 寻找替代方案 (关键！)
  → 卖出后的资金有更好的去处吗？
  → 替代方案预期收益 > 当前产品收益 + 交易成本？

Step 3: 计算净收益
  → 卖出手续费 + 买入手续费 + 预期收益差 = 净收益
  → 净收益 > 0 才值得操作

Step 4: 给出建议
  → 有明确更好去处 → 建议卖出并指定去向
  → 没有更好去处 → 建议不动，说明原因
  → 不确定 → 建议观察，给出观察指标
```

### 建议模板

```markdown
## 卖出建议

| 项目 | 当前产品 | 替代方案 |
|------|---------|---------|
| 年化收益 | X% | Y% |
| 交易成本 | 卖出费A + 买入费B | — |
| 净收益差 | Y% - X% - 交易成本 | — |

结论: 净收益差 > 0 → 值得操作 / 净收益差 ≤ 0 → 不动
```

## 验证方法

1. 每次建议卖出时，必须同时给出替代方案和净收益差计算
2. 如果没有替代方案，直接建议不动
3. "不动"不是被动选择，而是经过计算后的最优决策

## 教训总结

1. **卖出不是目的，换到更好的才是** — 没有更好去处就不动
2. **"不动"也是决策** — 经过计算后选择不动，比盲目操作更理性
3. **建议必须闭环** — 卖出→去向→净收益，缺一不可
